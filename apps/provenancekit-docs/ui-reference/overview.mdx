> `@provenancekit/ui` is a theme-aware React component library. Add a `<ProvenanceBadge>` to any piece of AI-generated content and users can instantly inspect its full provenance chain.

## Installation

<CodeGroup>
```bash pnpm
pnpm add @provenancekit/ui
```
```bash npm
npm install @provenancekit/ui
```
```bash yarn
yarn add @provenancekit/ui
```
</CodeGroup>

## Setup

### 1. Add the stylesheet

Import the design tokens in your global CSS file:

```css
/* globals.css */
@import "@provenancekit/ui/styles";
```

### 2. Wrap your app with the provider

```tsx
// app/layout.tsx (Next.js App Router)
import { ProvenanceKitProvider } from "@provenancekit/ui";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ProvenanceKitProvider
          apiUrl="https://api.provenancekit.com"
          apiKey={process.env.NEXT_PUBLIC_PK_API_KEY}
        >
          {children}
        </ProvenanceKitProvider>
      </body>
    </html>
  );
}
```

The provider creates an SDK instance that all child components use for data fetching.

### Provider props

| Prop | Type | Description |
|------|------|-------------|
| `pk` | `ProvenanceKit` | Pre-built SDK instance (takes priority over `apiUrl`/`apiKey`) |
| `apiUrl` | `string` | API base URL |
| `apiKey` | `string` | API key |
| `theme` | `ProvenanceKitTheme` | CSS custom property overrides |

## CSS Design Tokens

The library ships with a **light-first design system**. Dark mode is applied automatically when a `.dark` class is present on any ancestor element — typically `<html>` or `<body>` from your app's theme system.

Override any token globally or per-scope:

```css
:root {
  /* ── Node type colors ── */
  --pk-node-resource: oklch(0.52 0.18 250);  /* blue  */
  --pk-node-action:   oklch(0.50 0.18 145);  /* green */
  --pk-node-entity:   oklch(0.65 0.18 75);   /* amber */

  /* ── Entity role colors ── */
  --pk-role-human:    oklch(0.52 0.18 250);  /* blue   */
  --pk-role-ai:       oklch(0.52 0.2  310);  /* violet */
  --pk-role-org:      oklch(0.50 0.18 145);  /* green  */

  /* ── Badge (Pr squircle) ── */
  --pk-badge-bg:          oklch(0.13 0.04 250 / 92%);  /* near-black */
  --pk-badge-fg:          oklch(0.98 0 0);              /* white text */
  --pk-badge-font-family: 'Red Hat Display', system-ui, sans-serif;

  /* ── Graph canvas ── */
  --pk-graph-bg:             oklch(0.96 0.008 250);  /* light blue-gray canvas */
  --pk-graph-dot:            oklch(0.78 0.01 250);   /* dot grid */
  --pk-graph-node-bg:        oklch(1 0 0);           /* white node cards */
  --pk-graph-node-border:    oklch(0.88 0.01 250);
  --pk-graph-node-text:      oklch(0.13 0 0);
  --pk-graph-node-muted:     oklch(0.50 0 0);
  --pk-graph-control-bg:     oklch(1 0 0 / 92%);     /* controls backdrop */
  --pk-graph-control-border: oklch(0.88 0.01 250);
  --pk-graph-control-text:   oklch(0.50 0 0);

  /* ── Surfaces ── */
  --pk-surface:        oklch(1 0 0);
  --pk-surface-muted:  oklch(0.97 0.005 250);
  --pk-surface-border: oklch(0.90 0.005 250);
  --pk-foreground:     oklch(0.13 0 0);
  --pk-radius:         0.75rem;
}
```

<Note>
  The `--pk-badge-*` tokens stay dark in both light and dark mode by design — the "Pr" tag is a provenance mark that needs to stand out on any content.
</Note>

Or use the `theme` prop on `ProvenanceKitProvider` for the most common overrides:

```tsx
<ProvenanceKitProvider
  apiUrl="..."
  theme={{
    nodeResourceColor: "#6366f1",
    nodeActionColor:   "#22c55e",
    badgeBg:           "rgba(0,0,0,0.9)",
    radius:            "0.5rem",
  }}
>
  {children}
</ProvenanceKitProvider>
```

### Dark mode

Add the `.dark` class to your root `<html>` element (the standard Tailwind dark mode pattern):

```tsx
// With next-themes
import { ThemeProvider } from "next-themes";

<ThemeProvider attribute="class">
  <ProvenanceKitProvider apiUrl="...">
    {children}
  </ProvenanceKitProvider>
</ThemeProvider>
```

All components — cards, popovers, graph canvas, node cards, controls — will switch to dark surfaces automatically.

## Headless Mode

Every data-fetching component supports passing data directly — no network requests needed. This is useful for SSR, Storybook, and testing:

```tsx
// Pass nodes/edges directly — no auto-fetch
<ProvenanceGraph nodes={myNodes} edges={myEdges} />

// Pass a pre-fetched bundle
<ProvenanceBadge bundle={myBundle}>…</ProvenanceBadge>

// Pass a session object directly
<ProvenanceTracker session={mySession} />
```

## Next.js App Router pattern

Components are marked `"use client"` internally. In Next.js, re-export from a client boundary so Server Components can import them:

```tsx
// components/ui/pk-ui.tsx
"use client";
export {
  ProvenanceBadge,
  ProvenanceBundleView,
  ProvenanceGraph,
  ProvenanceTracker,
} from "@provenancekit/ui";
```

Then import from `components/ui/pk-ui` in any Server Component.

## Available Hooks

All data-fetching is available as standalone hooks:

```tsx
import {
  useProvenanceBundle,
  useProvenanceGraph,
  useSessionProvenance,
  useDistribution,
  useProvenanceKit,
} from "@provenancekit/ui";

// Access the configured SDK instance
const { pk } = useProvenanceKit();

// Fetch a provenance bundle
const { data: bundle, loading, error } = useProvenanceBundle("bafybei...");

// Fetch a provenance graph
const { data: graph } = useProvenanceGraph("bafybei...", { depth: 5 });

// Poll a session in real-time
const { data: session } = useSessionProvenance("session-xyz", { pollInterval: 3000 });

// Compute contribution weights from a bundle
const { data: distribution } = useDistribution("bafybei...");
```
